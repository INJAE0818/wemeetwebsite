# WEMEET 이력서 · 자소서 가이드

> 목표: WEMEET 확장 프로젝트를 삼성SDS / 클라우드 / 보안 / 금융 IT 직무 지원 문서에 자연스럽게 녹여낼 수 있도록, **기획 관점 + 문장 관점 + 이력서 뼈대**를 함께 정리한다.

---

## 1) 이 프로젝트를 이력서에 어떻게 포장해야 하나

핵심은 "퀴즈 서비스 만들었습니다"가 아닙니다.

다음과 같이 바뀌어야 합니다.

- 나쁜 표현: AI를 붙인 퀴즈 웹 서비스를 만들었습니다.
- 좋은 표현: 시험/자격증 학습 데이터를 구조화하고, 개인별 오답 패턴을 기반으로 반복학습과 연계 개념 탐색을 제공하는 AWS 기반 학습 플랫폼을 설계했습니다.

즉, 이 프로젝트는 아래 4축으로 보여줘야 합니다.

1. **문제 정의 능력**
2. **서비스/시스템 설계 능력**
3. **클라우드·보안 구현 능력**
4. **데이터/AI 활용 능력**

## 2-1) 이미 준비된 학습자료를 어떻게 말할까

사용자가 이미 `data/questions.json` 형태의 문제 덤프를 준비했다는 점은 큰 강점입니다.
이 부분은 반드시 **데이터를 직접 확보하고 구조화한 경험**으로 말해야 합니다.

추천 표현:
- 학습 플랫폼 MVP 검증을 위해 초기 문제 덤프를 직접 정리하고 JSON 기반 시드 데이터로 구조화했습니다.
- 향후 AWS SAA, 정보보안기사, 전산 7급 데이터까지 동일 스키마로 확장할 수 있도록 데이터 표준화 방향을 설계했습니다.
- 단순 프론트 기능 구현이 아니라, 문제 자산을 서비스 가능한 데이터 형태로 전환하는 작업까지 수행했습니다.

이 문장은 클라우드/백엔드/보안 직무에서 모두 유효합니다.
왜냐하면 실제 서비스는 결국 **데이터를 어떻게 다루느냐**가 핵심이기 때문입니다.

## 3) 이력서 bullet 작성 공식

가장 좋은 bullet은 아래 순서를 따릅니다.

> **무엇을 왜 만들었는지 → 어떻게 설계했는지 → 무엇이 좋아졌는지**

예시:
- 시험·자격증 학습 데이터를 구조화해 문제, 해설, 키워드, 관련 개념을 연결하는 데이터 모델을 설계하고, 오답 이후 연계 학습이 가능한 링크드 러닝 구조를 구현
- 사용자 정답/오답 이력을 기반으로 복습 우선순위를 계산하는 반복학습 로직을 설계해, 단발성 문제 풀이가 아닌 지속형 학습 경험 제공
- AWS 기반 인프라와 CI/CD 파이프라인을 설계해 학습 서비스의 배포 자동화와 운영 재현성을 강화
- JWT/RBAC, 감사 로그, 비밀정보 관리 구조를 반영해 학습 기록과 사용자 데이터의 접근 통제 및 추적 가능성 확보
- AI를 활용해 취약 개념 요약과 다음 학습 경로 추천 기능을 기획하고, 규칙 기반 데이터와 결합해 추천 신뢰성을 높이는 방향으로 설계

## 4) 자소서/지원서에 녹이는 방식

사용자가 작성하려는 문항은 사실상 아래를 묻고 있습니다.

1. 왜 이 분야에 지원하는가
2. 어떤 경험이 해당 분야와 연결되는가
3. 본인이 기획하고 실행한 프로젝트가 왜 적합한가

따라서 자소서는 **동기 → 문제의식 → 프로젝트 설계 → 직무 연결** 순서로 쓰면 좋습니다.

### 추천 전개 구조

#### 1문단 — 지원 동기
- 클라우드/보안/대규모 서비스에 대한 관심
- 단순 기능 구현보다 안정적 운영과 데이터 흐름 설계에 더 큰 흥미를 느꼈다는 점

#### 2문단 — 문제의식
- 기존 학습 서비스는 개인화 복습과 연계 개념 학습이 약하다고 느꼈음
- 단순 문제풀이형 서비스의 한계를 발견함

#### 3문단 — 프로젝트 기획
- AWS SAA, 전산 7급, 정보보안기사 등 실질적 시험 데이터를 대상으로 플랫폼을 확장하려는 이유
- Anki식 반복학습과 Obsidian식 링크 구조를 결합한 이유
- 여기에 AI 추천을 붙여 학습 효율을 높이려는 설계 방향

#### 4문단 — 기술/직무 연결
- AWS, 보안 통제, 운영 자동화, 데이터 모델링, AI 활용 역량이 지원 직무와 어떻게 연결되는지 설명

#### 5문단 — 입사 후 포부
- 단순히 기능 개발이 아니라, 신뢰성과 확장성을 갖춘 서비스 구축에 기여하고 싶다는 포부

## 5) 삼성SDS / 보안 / 금융 IT용 표현 차이

### 삼성SDS에 더 맞는 문장
- 서비스 기능 개발에 그치지 않고, AWS 기반 아키텍처와 운영 자동화를 함께 설계했습니다.
- 반복학습과 개념 연결 구조를 데이터 흐름 중심으로 모델링하며, 확장 가능한 플랫폼 형태를 고민했습니다.
- AI 기능을 단순 챗봇이 아니라 개인화 추천과 학습 경로 최적화에 적용해 서비스 가치로 연결했습니다.

### 보안 직무에 더 맞는 문장
- 사용자 학습 이력과 계정 데이터를 다루는 과정에서 인증, 권한 분리, 접근 추적, 비밀정보 관리 체계를 함께 설계했습니다.
- 추천/분석 기능에도 데이터 처리 기준과 감사 가능성을 반영해 신뢰 가능한 서비스 구조를 지향했습니다.

### 금융 IT/NH 계열에 더 맞는 문장
- 참여형 콘텐츠 구조가 사용자 리텐션을 높인다는 점에 주목했고, 이를 학습 서비스에 맞게 랭킹·리그·챌린지 구조로 재해석했습니다.
- 단순 정보 제공이 아니라, 사용자가 능동적으로 참여하고 성과를 체감하는 구조를 설계하고자 했습니다.

## 7) 실제 이력서 뼈대

### 헤드라인
클라우드 · 백엔드 · 보안 지향 개발자

### 소개 문장 예시
AWS 기반 서비스 설계와 보안 통제를 함께 고민하며, 데이터와 AI를 활용해 사용자 문제를 구조적으로 해결하는 백엔드 개발자입니다.

### 기술 스택 섹션
- **Backend**: Spring Boot / Node.js / REST API
- **Cloud**: AWS, Terraform, GitHub Actions, CloudWatch
- **Data**: MySQL/PostgreSQL, Redis
- **Security**: JWT, RBAC, Audit Log, Secret Management
- **AI/Data**: 추천 로직 설계, 학습 데이터 구조화, LLM 활용 기능 기획

### 대표 프로젝트 섹션
#### WEMEET Learn Platform
- 시험·자격증 학습 데이터를 구조화해 반복학습과 링크드 개념 탐색을 제공하는 AWS 기반 AI 학습 플랫폼 기획 및 개발
- 사용자 오답 이력 기반 복습 추천 로직과 문제-개념-키워드 연결 데이터 모델 설계
- 인증/인가, 감사 로그, 비밀정보 관리, 모니터링 체계를 포함한 운영형 서비스 구조 설계
- Terraform 및 CI/CD 기반 배포 자동화와 운영 재현성 확보

### 자격/준비중 섹션 예시
- AWS Certified Solutions Architect – Associate 준비 중
- 정보보안기사 준비 중
- 전산 7급 시험 준비 중

### 차별화 포인트 섹션
- 단순 문제풀이 앱이 아니라 데이터·보안·클라우드·AI를 함께 설계한 학습 플랫폼 경험
- 서비스 기획부터 백엔드, 운영 구조, 포트폴리오 문서화까지 연결 가능한 실행력

## 9) 마지막 조언
이 프로젝트는 잘만 만들면 "퀴즈 앱"이 아니라 아래처럼 보일 수 있습니다.

- 삼성SDS 관점: **클라우드 기반 AI 서비스 플랫폼 설계 경험**
- 보안 직무 관점: **인증·권한·감사·비밀관리까지 고려한 서비스 설계 경험**
- 금융 IT 관점: **참여형 서비스 구조와 데이터 기반 운영 설계 경험**

즉, 중요한 것은 기능 수가 아니라,
**문제 정의의 수준과 그것을 시스템 설계 언어로 바꾸는 방식**입니다.
