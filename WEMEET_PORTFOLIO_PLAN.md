# WEMEET 포트폴리오 기획서

> 목표: WEMEET를 단순한 웹 기능 프로젝트가 아니라, **AWS · 보안 · AI 기반 학습 플랫폼을 설계/운영할 수 있는 개발자**라는 점을 증명하는 핵심 포트폴리오로 재정의한다.

---

## 0) 프로젝트 재정의

기존 WEMEET가 "AI를 이용한 웹툰/퀴즈 도우미"였다면,
이제는 이를 다음과 같은 방향으로 확장해야 합니다.

### 새로운 프로젝트 정의
> **시험·자격증·직무 학습 데이터를 기반으로, AI가 개인별 약점을 분석하고 반복학습 경로를 추천하는 클라우드 기반 학습 플랫폼**

### 왜 이 방향이 좋은가
이 방향은 단순한 콘텐츠 서비스보다 훨씬 강하게 아래 역량을 보여줄 수 있습니다.

- **AWS 클라우드 설계 역량**
- **보안/인증/접근통제 설계 역량**
- **AI 활용 서비스 기획 역량**
- **데이터 모델링 및 추천 흐름 설계 역량**
- **운영 가능한 서비스 구조를 설계하는 백엔드 역량**

특히 삼성SDS, 안랩, NH투자증권 같은 조직을 노릴 때,
"AI를 썼다"보다 **AI를 실제 서비스 구조와 데이터 흐름 안에 어떻게 안전하게 붙였는가**가 더 중요합니다.

## 1) 핵심 비전

### 한 줄 소개
**Anki의 반복학습 + Obsidian의 링크드 지식구조 + AI 기반 약점 분석**을 결합한 학습 플랫폼.

### 사용자가 얻게 되는 가치
- 매일 개인별 복습 우선순위가 반영된 퀴즈 제공
- 틀린 문제에서 끝나지 않고, 관련 개념/키워드/연계 문제로 자동 확장
- 시험 합격에 필요한 취약 개념을 빠르게 보완
- 학습 이력을 기반으로 성장 경로를 시각적으로 확인

### 지원 직무와 연결되는 가치
- **삼성SDS 관점**: 대규모 데이터 흐름, 클라우드 아키텍처, 운영 자동화, 엔터프라이즈 서비스 확장성
- **보안 직무 관점**: 인증/인가, 민감 데이터 통제, 감사 로그, 비밀정보 관리, 접근 추적
- **금융 IT 관점**: 신뢰성, 규칙 기반 추천, 사용자 참여형 구조, 운영 지표 기반 의사결정

## 2) 서비스 컨셉 구체화

### 서비스명 예시
- WEMEET Learn Cloud
- WEMEET Memory Graph
- WEMEET StudyOps

### 문제 정의
많은 수험생 서비스는 문제 풀이 또는 강의 제공에 머무릅니다.
하지만 실제 합격을 좌우하는 것은 다음과 같습니다.

1. 내가 **어떤 개념에서 반복적으로 틀리는지** 파악되는가
2. 틀린 문제를 기준으로 **관련 개념까지 즉시 확장 학습**할 수 있는가
3. 복습 주기를 사람이 수동으로 관리하지 않아도 **자동으로 우선순위화**되는가
4. 장기적으로 학습량이 늘어나도 **클라우드 환경에서 안정적으로 운영**될 수 있는가

### 해결 아이디어
- 사용자가 문제를 풀면 AI가 오답 패턴을 분류
- 문제는 개념 노드, 키워드 노드, 시험 영역 노드와 연결
- 틀린 문제를 기준으로 관련 개념과 연계 문제를 트리/그래프 형태로 탐색
- 개인별 학습 이력과 정답률을 기반으로 다음 복습 세트를 자동 추천

## 3) 핵심 기능 기획

### 0. 데이터 자산 확보 현황
현재 저장소에는 `data/questions.json` 형태의 시드 문제 덤프가 이미 포함되어 있습니다.
이는 포트폴리오에서 매우 중요합니다. 이 프로젝트는 단순 기획이 아니라,
**실제 학습자료를 수집·구조화하고 서비스 데이터 자산으로 전환하기 시작한 프로젝트**라고 설명할 수 있기 때문입니다.

현재 데이터는 문제/선택지/정답 중심의 최소 구조이므로,
향후 `exam`, `domain`, `concepts`, `explanation`, `keywords` 필드를 추가해
시험별 학습 그래프와 추천 엔진 입력값으로 확장하는 흐름을 가져가면 좋습니다.

### 1. 시험별 문제 데이터 허브
대상 예시:
- AWS SAA
- 전산 7급
- 정보보안기사
- 추후 SQLD, 정보처리기사, 네트워크관리사 등으로 확장 가능

구성 요소:
- 문제
- 정답
- 해설
- 핵심 키워드
- 관련 개념
- 출제 영역
- 난이도
- 관련 문제 링크

### 2. Anki형 반복학습 엔진
- 정답/오답/망설임 여부를 바탕으로 복습 주기 계산
- 일일 학습 큐 자동 생성
- 취약 영역 우선 노출
- 복습 누적도와 장기 기억 유지율 추적

### 3. Obsidian형 링크드 러닝 그래프
- 문제 ↔ 키워드 ↔ 개념 ↔ 관련 문제를 그래프로 연결
- 사용자가 틀린 문제에서 연관 개념 문서로 즉시 이동
- 개념 페이지에서 다시 관련 문제로 이동 가능
- 트리형/그래프형 탐색 UI 제공

### 4. AI 학습 추천 기능
- 오답 이유 분류: 개념 부족 / 문제 해석 실패 / 단순 실수
- 취약 개념 요약 생성
- 다음에 풀 문제 세트 추천
- 특정 시험 목표 점수까지 남은 취약 영역 추정

### 5. 참여형 학습 기능
NH투자증권의 참여형 콘텐츠에서 착안할 수 있는 포인트는,
단순 정보 제공이 아니라 **사용자가 계속 돌아오게 만드는 구조**입니다.

이를 학습 서비스에 적용하면 다음과 같습니다.
- 주간 학습 리그
- 과목별 랭킹/배지 시스템
- 친구/스터디 그룹과 목표 공유
- 주간 승부예측처럼 "이번 주 달성률 예측" 챌린지
- 시즌별 학습 이벤트

즉, 금융 서비스의 게임화 요소를 "투자 재미"가 아니라
**학습 지속성**으로 치환하는 것이 핵심입니다.

## 4) 기술 포트폴리오로서의 강점 포인트

### A. AWS 역량을 보여주는 구조
추천 스택 예시:
- **Frontend**: React 또는 Next.js
- **Backend**: Spring Boot 또는 Node.js/NestJS
- **DB**: PostgreSQL 또는 MySQL
- **Cache**: Redis
- **Search**: OpenSearch 고려 가능
- **Object Storage**: S3
- **Infra**: Terraform
- **CI/CD**: GitHub Actions
- **Monitoring**: CloudWatch, X-Ray, Grafana 계열 검토 가능
- **Secrets**: AWS Secrets Manager / SSM

보여줘야 하는 포인트:
- IaC 기반 인프라 재현성
- 환경 분리(dev/stage/prod)
- 비밀정보 관리 체계
- 모니터링/알림 체계
- 비용/성능을 고려한 아키텍처 선택

### B. 보안 역량을 보여주는 구조
- JWT + Refresh 기반 인증
- RBAC 기반 권한 분리
- 관리자 기능 접근 통제
- HMAC 또는 민감 API 요청 무결성 검증
- 감사 로그 표준화
- Rate Limit / WAF 적용
- 개인정보/학습기록 접근 이력 추적
- OWASP Top 10 기반 점검표

### C. AI/데이터 역량을 보여주는 구조
- 문제/개념/키워드 그래프 데이터 모델링
- 개인별 오답 패턴 분석 파이프라인
- 추천 로직 설계
- AI 요약/분류 결과에 대한 품질 기준 정의
- 프롬프트 결과를 그대로 노출하지 않고 검증/가공하는 안전한 구조

## 5) 아키텍처 스토리텔링 포인트

### 한 장으로 설명할 구조
1. 사용자가 문제 풀이 요청
2. API 서버가 인증/권한 검증
3. 문제/개념/학습이력 데이터 조회
4. AI 추천 모듈이 다음 학습 세트 생성
5. 결과를 Redis 캐시 및 DB에 반영
6. 주요 이벤트는 감사 로그와 모니터링 지표로 저장

### 면접에서 강조할 말
- 단순히 추천 기능을 붙인 것이 아니라,
  **학습 이벤트가 데이터로 축적되고 그 데이터가 다시 추천 품질을 높이는 구조**로 설계했다.
- AI 결과를 맹신하지 않고,
  **문제·개념·오답 기록 기반의 규칙/데이터와 결합해 신뢰도 높은 추천**을 지향했다.

## 6) 포트폴리오 1페이지 초안

### 프로젝트 개요
- **프로젝트명**: WEMEET Learn Platform
- **한 줄 소개**: 시험·자격증 학습 데이터를 기반으로 반복학습과 링크드 개념 탐색을 지원하는 AI 학습 플랫폼
- **핵심 역할**: 서비스 기획, 백엔드/API 설계, AWS 인프라 설계, 보안 통제 설계, 학습 추천 로직 정의
- **타깃 도메인**: AWS SAA, 전산 7급, 정보보안기사

### 내가 해결하려는 문제
- 수험생은 틀린 문제를 다시 보는 수준에 머무르기 쉽고,
  실제로는 오답 뒤에 숨어 있는 개념 연결고리를 놓치기 쉽다.
- 또한 반복학습 우선순위를 수동으로 관리하면 장기 학습 효율이 떨어진다.
- 따라서 문제 풀이, 개념 연결, 복습 자동화, 개인화 추천을 하나의 플랫폼 안에서 해결하고자 했다.

### 핵심 구현 포인트
1. **반복학습 엔진**
   - 오답/정답 이력 기반 복습 주기 계산
   - 일일 추천 문제 세트 자동 생성
2. **링크드 개념 그래프**
   - 문제 ↔ 개념 ↔ 키워드 ↔ 연관 문제 연결
   - 오답에서 연계 학습으로 즉시 확장
3. **AI 추천 및 요약**
   - 취약 개념 분석
   - 개인별 다음 학습 경로 제안
4. **AWS 기반 운영 구조**
   - Terraform 기반 인프라 코드화
   - GitHub Actions 기반 배포 자동화
   - CloudWatch 중심 모니터링/알림
5. **보안 통제 체계**
   - 인증/인가 분리
   - 감사 로그 및 접근 추적
   - 비밀정보 안전 관리

### 기대 성과 지표 예시
- 사용자 주간 재방문율
- 오답 재발률 감소율
- 특정 시험 영역 정답률 향상
- 평균 복습 완료율 상승
- 배포 시간 및 장애 대응 시간 단축

### 회고 한 줄
> “WEMEET를 콘텐츠 중심 프로젝트에서, 데이터·AI·클라우드·보안 역량이 결합된 학습 플랫폼으로 확장해 실무형 포트폴리오로 발전시켰다.”

## 7) 구현 우선순위

### Phase 1 — 포트폴리오 MVP
- AWS SAA 한 개 시험 도메인부터 시작
- 문제/해설/키워드/개념 데이터 구조 설계
- 사용자 인증 및 학습 이력 저장
- 오답 기반 복습 큐 생성
- 기본 대시보드 구축

### Phase 2 — 차별화 기능
- 개념 그래프 연결
- AI 오답 원인 분류
- 취약 개념 추천
- 관리자용 데이터 적재/검수 도구

### Phase 3 — 운영형 서비스 고도화
- Terraform 전면 적용
- Secrets Manager 연동
- CloudWatch 경보 구축
- 감사 로그 대시보드
- 비용/성능 최적화

### Phase 4 — 지원용 포트폴리오 완성
- 아키텍처 다이어그램 정리
- 수치형 개선 지표 확보
- Notion 포트폴리오 제작
- 이력서/자소서/면접 답변 정합성 맞춤

## 8) 삼성SDS / 보안 / 클라우드 지원용 메시지 변환법

### 삼성SDS에 맞는 표현
- 대규모 사용자/데이터를 고려한 구조 설계
- AWS 환경에서의 안정적 운영과 자동화 경험
- 서비스 아키텍처와 데이터 흐름을 함께 설계한 경험
- 생성형 AI를 기능이 아닌 **업무/서비스 생산성 향상 도구**로 녹인 경험

### 보안 직무에 맞는 표현
- 학습 데이터와 사용자 이력에 대한 접근 통제 설계
- 감사 로그와 추적 가능성 확보
- 비밀정보 관리와 API 보호 체계 반영
- 추천/AI 기능에도 안전한 데이터 처리 원칙 적용

### 금융 IT 직무에 맞는 표현
- 사용자 신뢰성과 서비스 일관성 중심 설계
- 참여형 구조를 통한 사용자 리텐션 개선 아이디어
- 데이터 기반 추천과 지표 중심 운영 방식
- 이벤트/랭킹/리그형 요소를 통한 몰입도 설계

## 9) 최종 체크리스트
- [ ] 프로젝트가 단순 퀴즈 앱이 아니라 플랫폼처럼 설명되는가
- [ ] AWS 역량이 구체적 아키텍처 언어로 표현되는가
- [ ] 보안 설계가 로그인 기능 수준을 넘는가
- [ ] AI 활용이 단순 챗봇이 아니라 추천/분석 구조로 설명되는가
- [ ] 데이터 모델링 포인트가 드러나는가
- [ ] 자소서/이력서 문장으로 바로 변환 가능한가
